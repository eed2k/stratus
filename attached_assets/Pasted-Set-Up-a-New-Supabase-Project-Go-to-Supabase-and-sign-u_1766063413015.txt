Set Up a New Supabase Project
Go to Supabase and sign up/log in
Click "New Project"
Enter your project details:
Name: Stratus Weather
Database Password: 6103@EED (or generate a secure one)
Choose a region closest to you
Click "Create new project"
2. Configure Database Connection
After project creation, go to Project Settings > Database
Find the "Connection string" section
Copy the "Connection string (pooler)" - it should look like:
postgresql://postgres:[YOUR-PASSWORD]@db.xxxxxx.supabase.co:6543/postgres
3. Update Your .env File
Update your .env file with the new connection string:

bash
# Database Configuration
DATABASE_URL=postgresql://postgres:[YOUR-PASSWORD]@db.xxxxxx.supabase.co:6543/postgres
# Server Configuration
PORT=5000
NODE_ENV=development
# Enable demo mode
VITE_DEMO_MODE=true
4. Configure Database Access
In Supabase, go to Authentication > Policies
Click "Enable RLS" (Row Level Security) if you want to use it
Create necessary policies for your tables
5. Set Up Database Tables
You'll need to run your migrations. The project should have a migration file that sets up the necessary tables. Run:

bash
npm run db:push
6. Configure Network Access
In Supabase, go to Project Settings > Database
Under "Connection Settings", add your IP address to the allowed IPs
Or enable "Allow all IPs" for development (not recommended for production)
7. Start Your Application
bash
npm run dev
8. Initialize Demo Data
Once the server is running, in a new terminal:

bash
node scripts/init-demo.js
9. Access Your Application
Open your browser and go to: http://localhost:5000

Troubleshooting
Connection Issues:
Make sure your IP is whitelisted in Supabase
Check if your firewall allows outbound connections to port 6543
Try using a different network
Database Errors:
Make sure the database is fully provisioned (can take a few minutes)
Check the Supabase dashboard for any service issues
Environment Variables:
Ensure .env is in your project root
Restart your development server after changing environment variables
